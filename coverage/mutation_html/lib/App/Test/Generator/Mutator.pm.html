   26: =head1 DESCRIPTION
   27: 
   28: B<App::Test::Generator::Mutator> is a mutation engine that programmatically alters Perl source files to evaluate the effectiveness of a project's test suite.
   29: It analyzes modules, generates systematic code mutations (such as conditional inversions,
   30: logical operator changes,
   31: and other behavioral tweaks),
   32: and applies them within an isolated workspace so tests can be executed safely against each modified variant.
   33: By tracking which mutants are "killed" (cause tests to fail) versus those that "survive" (tests still pass),
   34: the module enables calculation of a mutation score,
   35: providing a quantitative measure of how well the test suite detects unintended behavioral changes.

  108: =head2 prepare_workspace
  109: 
  110: Prepares an isolated temporary workspace for a single mutation test run.
  111: 
  112: The entire C<lib/> tree is copied into the workspace so that all module
  113: dependencies resolve correctly when the test suite runs against the mutant.
  114: Only after this copy is complete is the single target file overwritten by
  115: C<apply_mutant>.
  116: 
  117: =head3 Arguments
  118: 
  119: Takes no arguments beyond the invocant.
  120: 
  121: =head3 Returns
  122: 
  123: A string containing the absolute path to the temporary directory that was created.
  124: The directory is automatically removed when the
  125: C<App::Test::Generator::Mutator> object goes out of scope (via
  126: L<File::Temp>'s C<CLEANUP =E<gt> 1> behaviour).
  127: 
  128: =head3 Side Effects
  129: 
  130: =over 4
  131: 
  132: =item *
  133: 
  134: Creates a temporary directory under the system's default temp location.
  135: 
  136: =item *
  137: 
  138: Recursively copies the entire C<lib_dir> tree (default C<lib/>) into the
  139: workspace using C<File::Copy::Recursive::dircopy>.
  140: 
  141: =item *
  142: 
  143: Sets C<< $self->{workspace} >> to the absolute path of the temporary directory.
  144: 
  145: =item *
  146: 
  147: Sets C<< $self->{relative} >> to the path of the target file relative to
  148: C<lib_dir>, for use by C<apply_mutant>.
  149: 
  150: =back
  151: 
  152: =head3 Notes
  153: 
  154: The workspace is only valid for a single file's mutation run.  Call
  155: C<prepare_workspace> once per file, then call C<apply_mutant> once per
  156: mutant within that file.  Because C<CLEANUP =E<gt> 1> is set, the workspace
  157: is silently removed when the tempdir handle is garbage collected - do not
  158: store the path beyond the lifetime of the enclosing scope.
  159: 
  160: =head3 Example
  161: 
  162:     my $mutator = App::Test::Generator::Mutator->new(
  163:         file    => 'lib/My/Module.pm',
  164:         lib_dir => 'lib',
  165:     );
  166: 
  167:     my @mutants = $mutator->generate_mutants();
  168: 
  169:     for my $mutant (@mutants) {
  170:         my $workspace = $mutator->prepare_workspace();
  171: 
  172:         $mutator->apply_mutant($mutant);
  173: 
  174:         local $ENV{PERL5LIB} = "$workspace/lib";
  175:         my $survived = (system('prove', 't') == 0);
  176: 
  177:         # workspace cleaned up automatically when $workspace goes out of scope
  178:     }

  209: 	my $tmp = tempdir(CLEANUP => 1);

  211: 	my $src = $self->{file};

  213: 	# Derive relative path automatically
